# 個人記帳 PWA — 第一階段測試版

不連接 Supabase，資料完全存在瀏覽器本機的 IndexedDB。用來驗證 PWA 基礎能力：可安裝、離線可開、重新整理資料不丟失、手機/桌面皆可用。

## 為什麼不能直接雙擊 index.html 打開？

Service Worker（離線快取的關鍵技術）基於瀏覽器安全限制，只在 `https://` 或 `localhost` 底下才會註冊生效。用 `file://` 直接開啟，網頁本身能動、IndexedDB 也能正常存資料，但**離線快取跟安裝提示不會出現**。所以測試時請用下面任一方式啟動本機伺服器。

## 本機測試方式

在這個資料夾底下，任選一種：

```bash
# 方式一：Python（大多數電腦都有）
python3 -m http.server 8080

# 方式二：Node.js
npx serve .
```

啟動後，用瀏覽器打開 `http://localhost:8080`。

## 測試檢查清單

- [ ] **安裝到桌面**：Chrome 網址列右側應出現「安裝」圖示（手機上是「加到主畫面」），點擊後桌面/主畫面會出現記帳app圖示
- [ ] **離線開啟**：正常打開一次後，把電腦網路關掉（或開發者工具勾選 Offline），重新整理頁面，畫面應該還能正常顯示
- [ ] **資料不丟失**：新增幾筆交易後重新整理頁面（或關掉再打開），資料應該還在
- [ ] **響應式**：把瀏覽器視窗縮小到手機寬度，畫面排版應該自動調整（底部導覽列變成手機常見的樣式）

## 目前完成的功能

- 總覽（本月支出/收入/結餘 + 近期交易）
- 交易列表（搜尋備註/分類關鍵字、依帳戶/分類篩選、刪除）
- 新增交易（支出/收入切換、對應不同分類選項）
- 分類管理（新增/刪除支出分類與收入分類）
- PWA：manifest.json + service worker 離線快取 + IndexedDB 本地儲存

## 目前刻意還沒做的部分（之後階段再處理）

- 沒有帳號登入，也沒有雲端同步——資料只存在「這一台裝置的這一個瀏覽器」，換裝置或清瀏覽器快取資料就會消失。這是下一步接 Supabase 後才會解決的事。
- 交易編輯功能（目前只能新增/刪除，還不能修改既有交易）
- 帳戶餘額計算（目前帳戶只是分類標籤，還沒有真的計算每個帳戶的實際餘額）
- CSV 匯出/匯入

## 檔案結構

```
expense-pwa/
├── index.html          畫面結構
├── styles.css           樣式（響應式）
├── app.js               IndexedDB邏輯 + 畫面渲染 + 表單邏輯
├── manifest.json         PWA設定檔（app名稱、圖示、啟動模式）
├── service-worker.js     離線快取邏輯
└── icons/                app圖示（192px、512px）
```

## 部署到 Vercel（測試安裝提示最準確的方式）

本機 `localhost` 也能觸發安裝提示，但如果想在手機上實際測試（尤其 iOS Safari 對 PWA 支援比較挑剔），最準的方式是部署到 Vercel 這類有 HTTPS 的地方：

1. 這個資料夾整個上傳到一個 GitHub repo
2. 到 vercel.com 用該 repo 建立專案，Framework Preset 選「Other」（純靜態網站，不需要建置指令）
3. 部署完成後用手機瀏覽器開啟 Vercel 給的網址，就能真正測試「加到主畫面」

等你確認這版測試沒問題，下一步就是把 IndexedDB 换成 Supabase 讀寫，並加上登入功能。
